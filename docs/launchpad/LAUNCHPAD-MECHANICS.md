## Обзор механик

Axiome Launch Suite предоставляет два основных типа запуска токенов:

| Тип | Описание | Для кого |
|-----|----------|----------|
| **Fair Launch** | Равные условия для всех, bonding curve | Мем-токены, community проекты |
| **Presale** | Раунды с различными условиями | Серьезные проекты с roadmap |

## Fair Launch

### Концепция

Fair Launch обеспечивает максимально честные условия запуска:
- **Нет presale** — все покупают по одной цене
- **Нет team allocation** — команда покупает наравне со всеми
- **Bonding Curve** — цена растет с каждой покупкой
- **Автоматическая ликвидность** — DEX листинг при достижении порога

### Bonding Curve

Используется модифицированная экспоненциальная кривая:

```
Price = BasePrice × (1 + k × Supply^n)

Где:
- BasePrice — начальная цена (устанавливается создателем)
- k — коэффициент роста (0.0001 - 0.001)
- Supply — текущий проданный supply
- n — степень кривизны (1.0 - 2.0)
```

#### Пример расчета

| Продано Supply | Множитель | Цена (при базе 0.0001 AXM) |
|----------------|-----------|---------------------------|
| 0% | 1.0x | 0.0001 AXM |
| 10% | 1.1x | 0.00011 AXM |
| 25% | 1.4x | 0.00014 AXM |
| 50% | 2.0x | 0.0002 AXM |
| 75% | 3.2x | 0.00032 AXM |
| 100% | 5.0x | 0.0005 AXM |

### Параметры Fair Launch

```typescript
interface FairLaunchConfig {
  // Основные параметры
  tokenName: string
  tokenSymbol: string
  totalSupply: number           // Общий supply токена

  // Bonding Curve
  basePrice: number             // Начальная цена в AXM
  curveCoefficient: number      // k: 0.0001 - 0.001
  curveExponent: number         // n: 1.0 - 2.0

  // Пороги
  softCap: number               // Минимум для успешного запуска (AXM)
  hardCap: number               // Максимум сбора (AXM)
  liquidityThreshold: number    // % от сбора на DEX ликвидность

  // Временные ограничения
  duration: number              // Длительность (часы)
  maxBuyPerWallet: number       // Лимит на кошелек (% от supply)

  // Anti-bot
  enableAntiBot: boolean        // Защита от ботов
  cooldownPeriod: number        // Задержка между покупками (секунды)
}
```

```

├── 80% → Ликвидность на AxiomeSwap (заблокирована)
├── 15% → Создателю проекта
├── 3%  → Treasury платформы
└── 2%  → Buyback LAUNCH
```

### Anti-Bot механизмы

1. **Cooldown Period** — задержка между транзакциями одного кошелька
2. **Max Buy Limit** — максимум покупки на кошелек
3. **Gradual Release** — постепенное открытие продажи
4. **Captcha Integration** — опциональная captcha для крупных покупок

## Presale (ICO)

### Концепция

Presale подходит для серьезных проектов с четким roadmap:
- **Несколько раундов** с разными условиями
- **Vesting** для ранних инвесторов
- **Whitelist** для держателей LAUNCH
- **KYC** опционально

## Anti-Rug механизмы

### Проблема Rug Pull

Типичные схемы мошенничества:
1. **Liquidity Pull** — создатель забирает ликвидность
2. **Honeypot** — токен нельзя продать
3. **Hidden Mint** — скрытая функция создания токенов
4. **Team Dump** — массовая продажа командой

### Решения Axiome Launch Suite

#### 1. Обязательный залог LAUNCH

```typescript
interface CreatorDeposit {
  // Размер залога зависит от типа проекта
  unverified: 0,              // Без залога (отмечается как high-risk)
  basicVerified: 1000,        // 1,000 LAUNCH
  premiumVerified: 5000,      // 5,000 LAUNCH
  featuredProject: 10000,     // 10,000 LAUNCH
}
```

**Условия возврата залога:**
- ✅ Успешный запуск + 30 дней без инцидентов → 100% возврат
- ⚠️ Soft cap не достигнут → 50% возврат
- ❌ Rug pull / мошенничество → 0% возврат (конфискация)

#### 2. Блокировка ликвидности

```
┌─────────────────────────────────────────────────────────────┐
│                   LIQUIDITY LOCK                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Создание LP токенов                                        │
│         ↓                                                   │
│  Отправка в Lock контракт                                   │
│         ↓                                                   │
│  Установка unlock time (мин. 6 месяцев)                     │
│         ↓                                                   │
│  LP заблокированы, создатель НЕ может их вывести            │
│         ↓                                                   │
│  После unlock time — постепенная разблокировка              │
│                                                             │
│  Параметры:                                                 │
│  • Минимальный lock: 6 месяцев                              │
│  • Для Featured: 12 месяцев                                 │
│  • Cliff period: первые 3 месяца без разблокировки          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 3. Anti-Whale лимиты

```typescript
interface AntiWhaleConfig {
  maxBuyPercent: number        // Макс. покупка (% от supply)
  maxWalletPercent: number     // Макс. баланс кошелька
  sellCooldown: number         // Задержка между продажами (часы)

  // Прогрессивный налог на крупные продажи
  sellTax: {
    tier1: { percent: 0, threshold: 0.1 },      // До 0.1% supply: 0%
    tier2: { percent: 2, threshold: 0.5 },      // 0.1-0.5%: 2%
    tier3: { percent: 5, threshold: 1.0 },      // 0.5-1%: 5%
    tier4: { percent: 10, threshold: Infinity } // >1%: 10%
  }
}
```

#### 4. Honeypot Detection

Автоматическая проверка контракта перед листингом:

```typescript
interface HoneypotCheck {
  // Статический анализ
  hasHiddenMint: boolean       // Скрытая функция mint
  hasBlacklist: boolean        // Функция блокировки адресов
  hasPausable: boolean         // Возможность остановки
  hasProxy: boolean            // Upgradeable proxy

  // Динамическая проверка
  canBuy: boolean              // Можно купить
  canSell: boolean             // Можно продать
  buyTax: number               // Налог на покупку
  sellTax: number              // Налог на продажу

  // Результат
  riskScore: 0-100             // 0 = безопасно, 100 = точно scam
  warnings: string[]           // Список предупреждений
}
```

#### 5. Team Token Lock

Токены команды блокируются с vesting:

```
Team Allocation (15% типично)
├── 0% при TGE (никаких токенов сразу)
├── 6 месяцев cliff (полная блокировка)
└── Линейная разблокировка 12-24 месяца
```

## Рейтинговая система

### Trust Score

Каждый проект получает Trust Score от 0 до 100:

```typescript
interface TrustScore {
  // Компоненты (веса)
  verification: number         // 25% — уровень верификации
  liquidity: number            // 20% — ликвидность и lock
  holders: number              // 15% — распределение холдеров
  activity: number             // 15% — активность торгов
  contract: number             // 15% — анализ контракта
  community: number            // 10% — активность сообщества

  // Итоговый балл
  total: number                // 0-100

  // Классификация
  rating: 'A' | 'B' | 'C' | 'D' | 'F'
}
```

### Расчет компонентов

```typescript
// Verification Score (0-25)
verificationScore = {
  unverified: 0,
  basic: 10,
  premium: 18,
  audited: 25
}

// Liquidity Score (0-20)
liquidityScore = min(20, (
  (liquidityUSD / 10000) * 5 +           // До 5 за размер
  (lockMonths / 12) * 10 +               // До 10 за длину lock
  (lpBurnedPercent / 100) * 5            // До 5 за burned LP
))

// Holders Score (0-15)
holdersScore = min(15, (
  (uniqueHolders / 500) * 7 +            // До 7 за количество
  (1 - top10Concentration) * 8           // До 8 за распределение
))

// Activity Score (0-15)
activityScore = min(15, (
  (daily_transactions / 100) * 5 +       // До 5 за транзакции
  (unique_traders / 50) * 5 +            // До 5 за уникальных трейдеров
  (volume_24h / market_cap) * 5          // До 5 за volume ratio
))

// Contract Score (0-15)
contractScore = 15 - (
  (hasHiddenMint ? 5 : 0) +
  (hasBlacklist ? 3 : 0) +
  (hasPausable ? 2 : 0) +
  (highSellTax ? 5 : 0)
)

// Community Score (0-10)
communityScore = min(10, (
  (telegramMembers / 1000) * 3 +
  (twitterFollowers / 5000) * 3 +
  (discordMembers / 500) * 2 +
  (socialEngagement) * 2
))
```

## Governance механизмы

### Project Voting

Держатели LAUNCH могут голосовать за проекты:

```typescript
interface ProjectVote {
  projectId: string

  // Типы голосования
  voteType: 'feature' | 'delist' | 'warning'

  // Feature vote — выдвижение в Featured
  feature: {
    quorum: 100000,            // 100k LAUNCH
    threshold: 0.66,           // 66% за
    duration: 72,              // 72 часа
  }

  // Delist vote — удаление проекта
  delist: {
    quorum: 500000,            // 500k LAUNCH
    threshold: 0.75,           // 75% за
    duration: 168,             // 7 дней
    evidence: string[],        // Требуются доказательства
  }

  // Warning vote — добавление предупреждения
  warning: {
    quorum: 50000,             // 50k LAUNCH
    threshold: 0.6,            // 60% за
    duration: 48,              // 48 часов
  }
}
```

## Интеграция с AxiomeSwap

### Автоматический листинг

После успешного Fair Launch или Presale:

```typescript
interface DEXListing {
  // Параметры пула
  pool: {
    token0: 'AXM',
    token1: projectToken,
    initialLiquidity: collectedAXM * 0.8,  // 80% от сбора
    initialPrice: listingPrice,
  }

  // Блокировка
  lpLock: {
    duration: 6 * 30 * 24 * 60 * 60,  // 6 месяцев в секундах
    beneficiary: creatorAddress,
    unlockSchedule: 'linear',          // Линейная разблокировка
  }

  // Уведомления
  notify: ['telegram', 'twitter', 'email']
}
```

## Заключение

Механики Axiome Launch Suite разработаны с учетом:

1. **Безопасности** — многоуровневая защита от мошенничества
2. **Справедливости** — равные возможности для участников
3. **Прозрачности** — все действия on-chain
4. **Устойчивости** — экономические стимулы для честного поведения

Комбинация технических и экономических механизмов создает среду, где:
- Создателям выгодно строить реальные проекты
- Инвесторам безопасно участвовать в запусках
- Сообщество контролирует качество проектов
- Платформа получает устойчивый доход
